# Workshop 2 Facilitator Guide (75 min)

**Title:** Systems & Automation — “Smart” Can Be Simple  
**Theme:** Systems are inputs + rules + outputs. Automation is rules running on repeat.  
**Audience:** Families with kids ~4–10  
**Format:** Parent + child required (no drop-off). Minimal screens. No open internet. No kid accounts.

---

## 1) Purpose & outcomes

### Big idea (kid-safe)
A **system** is a set of steps that turns **inputs** into **outputs**.  
Automation is when the steps happen **automatically**.

Kid version:
> “Systems are recipes for life.”

### What kids should be able to do by the end
- Explain (in their own words): **input → rule → output**.
- Identify at least 2 everyday systems at home/school.
- Build a simple “if/then” automation on paper.

### What parents should be able to do by the end
- Spot how systems shape behavior (especially around screens).
- Use the **3 questions** when deciding on tech features:
  1) What’s the input?
  2) What rule is running?
  3) What output/behavior does it create?
- Leave with a small **Family Automation Plan** (1–2 automations + 1 “anti-automation”).

---

## 2) Facilitator mindset

### Tone
- Curious, empowering. “We can design our environment.”
- Avoid making it about surveillance. Focus on **reducing friction and arguments**.

### Redirection script (when parents spiral)
> “Let’s translate it into system language. What’s the input? What’s the rule? What output is it producing? Then we can change one part.”

---

## 3) Materials & prep

### Printables
1) **Input / Rule / Output** cards (1 set per family) *(or write on index cards)*
2) **If / Then** strips (1 sheet per family)
3) **Family Automation Plan** handout (1 per family)

### Supplies
- Sticky notes
- Markers + pens
- Index cards (backup)
- Painter’s tape
- Poster paper or whiteboard
- Timer

### Optional props
- A simple “system object” like:
  - a kitchen timer
  - a light switch (or picture)
  - a thermostat photo
  - a calendar page

### Room setup
- Tables grouped by families.
- Board space for:
  - “System = Input → Rule → Output”
  - A big example: “IF ___ THEN ___”

### Board plan (before start)
1) **Workshop 2: Systems & Automation**
2) System model: **Input → Rule → Output**
3) Automation language: **IF (input) THEN (output)**
4) Norms: kind, curious, no shaming

---

## 4) Safety / boundaries
- Kids stay with their adult.
- Minimal screens.
- No photographing other families without permission.

## 5) Run of show (minute-by-minute)

### 0–5 min — Welcome + quick recap
**Goal:** reconnect to Workshop 1 and set expectations.

Script:
> “Welcome back. Last week: tech is tools made by people for a job. This week: tools are often part of a **system**.”
> 
> “Parent + child together. Kind, curious, no shaming.”

Quick warm-up (30 seconds):
- “Point to your Tech Values Card on your fridge—did anyone use it once?”
  - Celebrate any win; keep moving.

### 5–15 min — Teach the system model (Input → Rule → Output)
**Goal:** give kids a simple mental model.

Teach (2–3 min):
- **Input:** what the system notices (button press, time of day, a notification)
- **Rule:** what it does with that input (“if X, then Y”)
- **Output:** what happens (sound plays, video starts, message appears)

Call-and-response:
- “Input!” (kids: “What it notices!”)
- “Rule!” (kids: “What it decides!”)
- “Output!” (kids: “What happens!”)

Whole-room examples (5 min):
- Light switch:
  - Input: flip switch
  - Rule: if up → on
  - Output: light
- Bedtime routine:
  - Input: it’s 7:30pm
  - Rule: brush teeth → pajamas → book
  - Output: ready for bed

Family prompt (3–4 min):
> “Name one system at your house. What’s the input, rule, output?”

Share 2 examples.

### 15–35 min — Activity 1: Build a paper automation (If/Then)
**Goal:** families create silly + real automations and practice the model.

Instructions (2 min):
> “At your table, you’ll build 2 ‘If/Then’ automations: one silly, one real. Use the strips/cards.”

Examples to put on the board:
- Silly: IF someone says “banana” THEN everyone freezes.
- Home: IF it’s 7:30pm THEN start bedtime routine.
- School: IF the bell rings THEN line up.

Work time (10–12 min):
- Walk around.
- Coach parents to let kids lead one automation.

Mini-share (8–10 min):
- 3 families share their silly automation (quick laughs).
- 2 families share their real automation.

Facilitator point (1 min):
> “Automation isn’t magic—it’s just rules running on repeat.”

### 35–55 min — Activity 2: Spot the automation in screen tools
**Goal:** connect system model to real tech behaviors without using devices.

Set-up (1 min):
> “Now we’ll translate a few common app/device behaviors into system language.”

Choose 3–4 examples (write them on the board):
- Autoplay
- Notifications
- Recommended videos
- Streaks / badges

For each, ask the room:
1) Input: what triggers it?
2) Rule: what does the app do?
3) Output: what happens next?
4) Family impact: how does that change behavior?

Example (Autoplay):
- Input: video ends
- Rule: start next video after countdown
- Output: kid keeps watching

Parent takeaway:
> “If you don’t like the output, change the input or the rule.”

### 55–70 min — Family deliverable: Automation Plan
**Goal:** leave with 1–2 helpful automations + 1 intentional “anti-automation.”

Introduce (1 min):
> “We’re going to design our environment so it’s easier to win.”

Families fill out:
1) **One automation we want** (reduce friction)
2) **One automation we want to turn off** (reduce pull)
3) **One default rule** (when unsure)

Give options to spark ideas:
- Want:
  - IF it’s homework time THEN devices park in a basket
  - IF we get home THEN shoes + backpacks go to one spot
  - IF it’s 7:30pm THEN start bedtime routine
- Turn off:
  - Autoplay
  - Notifications after 7pm
  - “Up next” recommendations
- Default:
  - “Ask a parent before changing settings.”

Share-out (optional): 2 families share one automation.

### 70–75 min — Close + Family Move
Script:
> “Family Move: pick ONE automation from your plan and try it once this week.”
> 
> “Next week: data—what gets collected, what accounts do, and how defaults shape privacy.”

## 6) Common scenarios & handling

### A) Parent asks: “So should we ban autoplay/notifications?”
> “Not always. Decide based on the job you want the tool to do. If autoplay breaks the job, turn it off.”

### B) Kid gets stuck on ‘rules’ concept
- Use a “recipe” analogy.
- Ask: “If you press play, what happens?”

### C) Parent goes into deep tech talk
> “Love it. For today we’ll stay at the family level: input, rule, output—enough to make good decisions.”

## Appendix A — Suggested “If/Then” prompts (kid-friendly)

- IF it’s dinner time THEN devices go to the charger.
- IF we’re in the car THEN we pick music together.
- IF someone is talking THEN we pause the TV.
- IF it’s Saturday morning THEN we watch one show (then stop).
- IF we want a new app THEN we ask a parent.

## Appendix B — “Anti-automation” ideas (reduce pull)

- Turn off autoplay
- Turn off “Up next” previews
- Disable non-essential notifications
- Remove app icons from the home screen
- Create a “parking spot” for devices during meals/bedtime
